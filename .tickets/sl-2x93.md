# arrows: undocumented nl-derived classification not in CLI docs or spec

The arrows command emits an 'nl-derived' classification for implicit arrows inferred from NL backtick references, but this value is not documented anywhere.

SATSUMA-CLI.md (Transform Classification section) documents exactly four values:
  structural, nl, mixed, none

The arrows command returns a fifth value 'nl-derived' for arrows created from NL backtick references (e.g., when a derived field references source fields in NL text).

What I did:
  satsuma arrows source_sys.name --json /tmp/satsuma-test-arrows/all-arrows.stm

What I expected:
  classification to be one of the four documented values

What actually happened:
  { "classification": "nl-derived", "transform_raw": "(NL ref)", "derived": true, ... }

The 'nl-derived' classification was added in ticket sc-akx6 but the CLI docs and AI-AGENT-REFERENCE.md were not updated to describe it.

This matters for agents that switch on classification values — they will hit an unexpected case.

Reproducer file: /tmp/satsuma-test-arrows/all-arrows.stm
