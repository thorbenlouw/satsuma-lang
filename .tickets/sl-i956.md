# schema: text output omits comments (// //! //?) from schema body

All three comment types (//, //!, //?) are stripped from the schema text output. While stripping // info comments is arguably acceptable, //! (warning) and //? (question/TODO) comments carry semantic information that is useful context when viewing a schema definition.

**What I did:**
```bash
satsuma schema comment_test /tmp/satsuma-test-schema/comments-test.stm
```

**Source:**
```
schema comment_test {
  id       INT      (pk)
  // This is an info comment
  name     STRING   (required)
  //! This is a warning about data quality
  email    STRING   (pii)
  //? Should this field be encrypted?
  password STRING
}
```

**Actual output:**
```
schema comment_test {
  id                      INT (pk)
  name                    STRING (required)
  email                   STRING (pii)
  password                STRING
}
```

All comments are stripped. This is also visible on `examples/sfdc_to_snowflake.stm` where the `sfdc_opportunity` schema has `//!` warnings on `CUST_TYPE` and `EMAIL_ADDR` that are lost.

Compare with the existing ticket sl-c1he which reports the same issue for `satsuma metric`.

**Reproducer:** `/tmp/satsuma-test-schema/comments-test.stm`, also `examples/db-to-db.stm` (legacy_sqlserver schema).
