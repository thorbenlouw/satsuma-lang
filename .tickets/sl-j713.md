# consistency: graph metric nodes missing fields array that metric command returns

Commands: satsuma graph --json vs satsuma metric --json

Graph nodes for metrics (kind='metric') do not include a 'fields' array, while schema nodes (kind='schema') do include fields. However, the metric command returns fields for metrics.

Example — metrics-platform/monthly_recurring_revenue:
  satsuma metric monthly_recurring_revenue --json -> fields: [{name: 'value', type: 'DECIMAL(14,2)', ...}]
  satsuma graph --json -> metric node has no 'fields' key at all

All 7 metrics in the metrics-platform example are affected:
  monthly_recurring_revenue: metric has 1 field, graph has 0
  churn_rate: metric has 1 field, graph has 0
  customer_lifetime_value: metric has 3 fields, graph has 0
  conversion_rate: metric has 1 field, graph has 0
  pipeline_value: metric has 2 fields, graph has 0
  order_revenue: metric has 5 fields, graph has 0
  cart_abandonment_rate: metric has 1 field, graph has 0

The graph also has no field-level edges (edges array) for metrics — only schema_edges with metric_source role.

A consumer using the graph for one-shot workspace reasoning (as recommended in CLI docs) would miss all metric field information.
