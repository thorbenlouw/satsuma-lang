# meta: cannot find fields from fragment spreads in consuming schema

When a schema uses a fragment spread (`...fragment_name`), fields from the fragment are not findable by `satsuma meta schema.field`.

What I did:
  `satsuma meta uses_fragment.created_at /tmp/satsuma-test-meta/spread-meta.stm`

The fixture defines:
  fragment tagged_fragment {
    created_at TIMESTAMPTZ (required, note "Creation timestamp")
    updated_at TIMESTAMPTZ
  }

  schema uses_fragment {
    id    INT (pk)
    name  STRING
    ...tagged_fragment
  }

Expected:
  Metadata for 'uses_fragment.created_at':
    type: TIMESTAMPTZ
    [tag] required
    note: Creation timestamp

Actual output:
  Field 'created_at' not found in schema 'uses_fragment'.
  Exit code: 1

Root cause: The field lookup in meta.ts uses `schema.fields` from the index, which does not expand fragment spreads. The CST-level `findFieldDecls` function also only matches `field_decl` nodes, not `fragment_spread` nodes, so it cannot traverse into the fragment's body.

Related tickets: sl-3aff (fields command, closed), sl-zjec (schema --json, closed) — those were fixed for their respective commands but the same issue persists in meta.
