Validator does not detect when a metric's source metadata references a schema that does not exist in the workspace.

**What I did:**
Created a file with a metric referencing `nonexistent_schema` as its source. Ran:
  satsuma validate /tmp/satsuma-test-validate/metric-bad-source.stm

**Expected:** Warning or error about 'nonexistent_schema' being undefined.

**Actual:** 'Validated 1 file: no issues found.' with exit code 0.

**Reproducer:** /tmp/satsuma-test-validate/metric-bad-source.stm
