# find --json: note metadata truncated to bare tag name in metadata array

In find --json output, note key-value pairs in the metadata array lose their value text.

Repro:
  satsuma find --tag classification examples/filter-flatten-governance.stm --json
  (inspect the consent_flags entry)

Expected: metadata array includes full note text: note "e.g. ['email_marketing', 'sms', 'third_party']"
Actual: note rendered as bare 'note' (value dropped):
  metadata: ['classification "RESTRICTED"', 'retention "7y"', 'note']

Other KV pairs like classification and retention render correctly with their values.
