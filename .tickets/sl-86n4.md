# summary: --compact flag has no effect on --json output

When combining --compact and --json flags, the output is identical to --json alone. The --compact flag is a no-op for JSON output.

What I did:
  diff <(satsuma summary examples/ --json) <(satsuma summary examples/ --compact --json)

What I expected:
  Compact JSON should omit notes, NL strings, or reduce detail compared to plain JSON.

What actually happened:
  Both commands produce byte-identical output. --compact has zero effect on JSON.

Other commands like 'satsuma schema' do apply --compact to JSON output (e.g. stripping metadata from fields). The summary command does not.

Repro: satsuma summary examples/ --compact --json vs satsuma summary examples/ --json
