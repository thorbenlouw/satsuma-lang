# mapping: mapping-level metadata block dropped from text and JSON output

Metadata blocks on the mapping definition itself (between the name and the body) are silently dropped from both text and JSON output.

**What I did:**
Created a mapping with block-level metadata:
```stm
mapping 'metadata test' (
  note "This mapping has block-level metadata"
  scd type2
  priority high
) {
  source { \`meta_src\` }
  target { \`meta_tgt\` }
  id -> id
  val -> result { trim }
}
```

```bash
satsuma mapping 'metadata test' /tmp/satsuma-test-mapping/mapping-meta.stm
```

**Expected:** Text output shows the metadata block. JSON output includes a metadata field.

**Actual text:**
```
mapping 'metadata test' {
  source { \`meta_src\` }
  target { \`meta_tgt\` }
  id -> id
  val -> result { trim }
}
```
The `(note "...", scd type2, priority high)` metadata is completely absent.

**Actual JSON:** No metadata field at all in the JSON object.

**Test file:** /tmp/satsuma-test-mapping/mapping-meta.stm
