# fields: text output does not show list_of for scalar list fields

Scalar list fields (list_of STRING, list_of INT) display as bare types in fields text output, making them indistinguishable from scalar fields.

Repro:
  satsuma fields order_events examples/filter-flatten-governance.stm

Expected: promo_codes shows as 'list_of STRING' (or similar list indicator).
Actual: promo_codes shows as 'STRING', tag_ids shows as 'INT'.

The schema text command correctly renders 'list_of STRING'. The fields --json output correctly includes isList: true. Only the fields text rendering is lossy.
