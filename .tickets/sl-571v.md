# nl: metric description string not included in nl output

The metric description string (the string literal between the metric name and its metadata parens) is not returned by 'satsuma nl <metric>'.

Reproduction:
  metric monthly_revenue "Total monthly revenue" (source s, grain monthly) {
    note { "Body note" }
  }
  satsuma nl monthly_revenue
  # Returns only the body note, not the description string

  satsuma metric monthly_revenue --json
  # Shows displayName: "Total monthly revenue" — so the parser has it

Expected: The description string is NL content on the metric and should appear in nl output, similar to how mapping-level and schema-level notes appear. It would be kind: "note" with parent: "monthly_revenue".

The metric description string IS accessible via 'satsuma metric --json' (as displayName) but is invisible to the nl extraction pathway, which is the canonical way agents discover NL content in a workspace.
