# validate/lint: lint exits 2 for warning-only findings, inconsistent with validate exit 0

When lint produces only warning-severity findings (e.g., unresolved-nl-ref), it exits with code 2. Meanwhile, validate exits 0 for warning-only findings (e.g., undefined-ref warnings). This is inconsistent.

Repro:
```bash
# Lint: warning-only -> exit 2
satsuma lint test.stm  # unresolved-nl-ref (warning) -> EXIT 2

# Validate: warning-only -> exit 0
satsuma validate test.stm  # undefined-ref (warning) -> EXIT 0
```

The CLI exit code convention documented in SATSUMA-CLI.md says: 0 = success, 1 = not found, 2 = parse error or filesystem error. Warnings should not trigger exit 2. Lint should exit 0 for warnings-only and exit 2 only for error-severity findings.
