# consistency: where-used nl_ref names include internal scope prefix 'note:metric:'

Command: satsuma where-used <schema> --json (in metrics-platform example)

When a schema is referenced via @ref in a metric's note block, the where-used command reports it as an nl_ref with a name like 'note:metric:churn_rate' rather than just 'churn_rate'.

Example:
  satsuma where-used fact_subscriptions examples/metrics-platform --json

Returns refs including:
  { kind: 'nl_ref', name: 'note:metric:churn_rate', row: 52 }
  { kind: 'nl_ref', name: 'note:metric:churn_rate', row: 53 }

Issues:
1. The 'note:metric:' prefix is an internal scope marker leaking into the JSON output. Other nl_ref entries use clean mapping names (e.g., 'opportunity ingestion').
2. These nl_ref entries from metric notes do not have corresponding edges in graph --json schema_edges (which only has metric_source edges, not nl_ref edges from metric notes).
3. A consumer processing where-used refs would not expect to parse internal prefixes from the name field.

Affected: examples/metrics-platform (fact_subscriptions, fact_orders).


## Notes

**2026-03-31T12:05:06Z**

Cause: nlRef.mapping carried internal scope prefixes (note:metric:, note:schema:, etc.) from extractStandaloneNoteRefs into the where-used JSON output.
Fix: Added stripInternalScopePrefix() in where-used.ts that removes the internal prefixes before emitting the name field. Standalone file-level notes get a descriptive placeholder '(file-level note)'.
