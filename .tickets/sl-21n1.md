# schema: text output drops backticks from backtick-quoted field identifiers

When a schema has fields with backtick-quoted identifiers (e.g., \`field-with-dashes\`), the text output strips the backticks. This makes the output not round-trippable as valid Satsuma — identifiers containing special characters (hyphens, dots) need backticks to be valid.

**What I did:**
```bash
satsuma schema backtick_test /tmp/satsuma-test-schema/backtick-fields.stm
```

**Expected:**
```
\`field-with-dashes\`       STRING(100) (required)
\`field.with.dots\`         INT
```

**Actual:**
```
field-with-dashes       STRING(100) (required)
field.with.dots         INT
```

The backticks are stripped. Per spec section 2.2, backtick identifiers are used for "identifiers containing special characters (spaces, dots, hyphens)" — stripping them makes the field names ambiguous (e.g., `field.with.dots` could be misread as a dotted path).

The JSON output similarly strips backticks from the `name` field.

**Reproducer:** `/tmp/satsuma-test-schema/backtick-fields.stm`
