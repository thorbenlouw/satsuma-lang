# vscode: upgrade showLineage to schema DAG webview (replace output channel)

Replace the output channel rendering in src/commands/lineage.ts with a new SchemaLineagePanel webview. The panel calls 'satsuma lineage --from <schema> --json' (or --to) and renders the resulting DAG as schema name pills connected by directed edges with mapping name labels. Nodes are coloured by kind: schema=orange pill, mapping=charcoal pill. Use ELK layered layout (same direction=RIGHT strategy as the field lineage panel). Keep the schema picker quickpick for selecting the root schema.

## Acceptance Criteria

- showLineage opens a webview panel instead of the output channel
- Schema nodes rendered as orange pills, mapping nodes as charcoal pills
- Directed edges labelled with the mapping name
- ELK layered layout used for positioning
- Click on a schema pill navigates to its declaration in the editor
- Click on a mapping pill navigates to its declaration
- Dark/light theme applied correctly
- Output channel fallback removed; old src/commands/lineage.ts output channel code deleted
