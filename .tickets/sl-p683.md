# context: does not search inner note blocks (note {} inside metric/mapping) for query matches

The `satsuma context` command does not search the text of note blocks that appear inside metric or mapping blocks. Schema-level `(note "...")` metadata IS searched, but `note { "..." }` blocks nested inside metrics/mappings and standalone file-level note blocks are not.

**What I did:**
```
satsuma context "transaction" /tmp/satsuma-test-context/ --json
satsuma context "calendar" /tmp/satsuma-test-context/ --json  
satsuma context "completed" /tmp/satsuma-test-context/ --json
satsuma context "ingestion" /tmp/satsuma-test-context/ --json
satsuma context "syncs" /tmp/satsuma-test-context/ --json
```

**Expected:** 
- 'transaction' should match monthly_active_customers metric (note says 'Count of customers with at least one transaction in the calendar month')
- 'completed' should match total_revenue metric (note says 'Sum of total_amount for completed orders')
- 'ingestion' should match the orders.stm standalone note block ('Order processing pipeline — handles order ingestion and transformation')
- 'syncs' should match the customers.stm standalone note block ('Customer data pipeline — syncs CRM customers to the data warehouse')

**Actual output:** All queries return `[]` (empty results).

**Root cause:** The workspace index only stores schema/mapping-level `note` metadata in `entry.note`. Inner note blocks (`note { "..." }` inside metrics/mappings) and standalone file-level note blocks are not indexed. The `scoreEntry` function only checks `entry.note` which covers `(note "...")` schema/mapping metadata but not block-level notes.

**Repro files:** /tmp/satsuma-test-context/customers.stm (standalone note block, metric with inner note), /tmp/satsuma-test-context/orders.stm (standalone note block, metric with inner note)
