# validate: undefined transform spread references not caught

Validator does not detect when a mapping arrow uses a transform spread referencing a transform that does not exist. The file validates cleanly with exit code 0.

**What I did:**
Created a file with `...nonexistent_xform` inside a mapping arrow body. Ran:
  satsuma validate /tmp/satsuma-test-validate/transform-spread.stm

**Expected:** Warning or error about 'nonexistent_xform' being undefined.

**Actual:** 'Validated 1 file: no issues found.' with exit code 0.

**Reproducer:** /tmp/satsuma-test-validate/transform-spread.stm


## Notes

**2026-03-22T02:00:00Z**

Cause: Validator only checked fragment spreads, not transform spreads in arrow pipe chains.
Fix: Walk arrow steps for fragment_spread entries and resolve against index.transforms (commit b79e255).
