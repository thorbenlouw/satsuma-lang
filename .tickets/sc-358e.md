# vscode: add field lineage affordance to sz-schema-card field rows in viz panel

In tooling/satsuma-viz/src/components/sz-schema-card.ts: on field row hover, show a small lineage icon (SVG, matching the existing icon style) at the right edge of the row. Clicking it dispatches a new 'sz-field-lineage' CustomEvent with detail { fieldPath: '<schemaId>.<fieldName>', schemaId }. In tooling/vscode-satsuma/src/webview/viz/viz.ts: listen for sz-field-lineage events on the vizEl and post { type: 'traceFieldLineage', fieldPath } to the extension host. In tooling/vscode-satsuma/src/webview/viz/panel.ts: handle the traceFieldLineage message by calling vscode.commands.executeCommand('satsuma.traceFieldLineage', { fieldPath }).

## Acceptance Criteria

- Hovering a field row in the viz panel reveals the lineage icon (hidden when not hovered)
- Icon does not appear on schema card headers, only on field rows
- Clicking the icon opens/focuses the FieldLineagePanel for that field
- The viz panel remains focused after the action (does not steal focus from the lineage panel)
- Icon is accessible: has aria-label='Trace field lineage for <fieldName>'
- sz-schema-card exports SzFieldLineageEvent type alongside existing event types
- The icon is visually coherent: uses --sz-text-muted colour, brightens to --sz-orange on hover
