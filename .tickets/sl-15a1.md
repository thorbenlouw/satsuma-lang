# arrows --as-source returns arrows where field is target, not source

The `--as-source` flag on `arrows` is supposed to filter to arrows where the queried field appears as the source (left side of `->`). But it returns arrows where the field appears as the TARGET.

Repro:
```bash
satsuma arrows regulatory_report.amount bug-hunt/ --as-source --json
# Returns:
# {source: 'ledger_entry.amount', target: 'regulatory_report.amount', ...}
# regulatory_report.amount is the TARGET, not the source!
# Expected: No results (regulatory_report.amount is never a source)
```

Contrast with `ledger_entry.amount --as-source` which correctly returns 3 arrows where `ledger_entry.amount` IS the source.

The bug appears to be that `--as-source` still matches when the queried field is on the target side of an arrow.

## Acceptance Criteria

1. `--as-source` only returns arrows where the queried field is the source (left side of ->)
2. `--as-target` only returns arrows where the queried field is the target (right side of ->)
3. Without either flag, both source and target matches are returned
4. Test with fields that appear on both sides of arrows in different mappings
