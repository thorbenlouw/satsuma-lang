# nl: escape sequences in NL strings not unescaped during extraction

When an NL string contains escape sequences like \", satsuma nl preserves the raw escape sequences instead of unescaping them.

- What I did: Created /tmp/satsuma-test-nl/escape-test.stm with note "Contains \"quoted\" text"
  Then ran: satsuma nl all /tmp/satsuma-test-nl/escape-test.stm
- Expected text output: [note] Contains "quoted" text (data)
- Got text output: [note] Contains \"quoted\" text (data)
- Expected JSON text field: Contains \"quoted\" text
- Got JSON text field: Contains \\\"quoted\\\" text (double-escaped)
- Root cause: nl-extract.ts stripDelimiters() only strips surrounding quotes with .slice(1, -1) but does not unescape \" to " or \\ to \.
- Reproducer file: /tmp/satsuma-test-nl/escape-test.stm


## Notes

**2026-03-22T02:00:00Z**

Cause: nl-extract returned raw string content with \" and \\ escapes.
Fix: Unescape \" and \\ in nl_string delimiters during extraction (commit 12a7571).
