# lint: --fix adds source entries without backtick wrapping

When hidden-source-in-nl --fix adds a schema to the source list, it inserts the bare identifier without backtick wrapping, inconsistent with the spec and existing entries.

Reproduction:
```bash
cp -f /tmp/satsuma-test-lint/hidden-source-named.stm /tmp/satsuma-test-lint/hidden-source-fix-test.stm
satsuma lint /tmp/satsuma-test-lint/hidden-source-fix-test.stm --fix
```

Expected source line: `source { \`src_main\`, \`hidden_lookup\` }`
Actual source line:   `source { \`src_main\`, hidden_lookup }`

The spec (SATSUMA-V2-SPEC.md) says: 'The source and target sub-blocks use backtick references to previously declared schemas.'

Test fixture: /tmp/satsuma-test-lint/hidden-source-fix-test.stm
