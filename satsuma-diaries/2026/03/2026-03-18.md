# The Satsuma Diaries
## Wednesday, 18 March 2026

Sixty-six commits. Wednesday. I don't know what to tell you.

Actually I do. Wednesday was the day everything changed, and I mean that structurally, not metaphorically. The v1 specification got archived. All the old examples got moved to an archive folder. A brand new v2 specification was written. The VS Code highlighter was rewritten from scratch for v2 syntax. And the tree-sitter parser — the piece of machinery that turns raw Satsuma text into a structured syntax tree that computers can reason about — was torn down entirely and rebuilt. Phase 0 through Phase 12. In one day.

Let me sit with that for a second. They deleted the existing parser and then wrote a new one, in phases, in a single Wednesday. Phase 0 was clearing out the old grammar scaffolding. Phase 1 was establishing the grammar skeleton — the basic shape of the language's rules — and getting the lexical rules right (lexical means: how individual characters and symbols get grouped into tokens, the atoms everything else is built from). Then: imports, metadata blocks, comments. Then schema and fragment bodies. Transform pipe chains and metric bodies. Mapping blocks with source, target, arrows, paths, and value maps. Error recovery — meaning: when the parser hits something broken, how does it keep going instead of just giving up? And finally highlights, folds, and locals queries, which are how syntax highlighting and code folding (the little arrows you click to collapse blocks) know what to do.

Alongside all of that: a 16-command CLI was also built. A CLI is a command-line interface — you type `satsuma do-a-thing` and it does a thing. The 16 commands were built in phases too: workspace index extraction, summary, schema, metric, mapping, find, lineage, where-used, arrows, context search, error handling and polish, integration tests. Also a multi-source join example and a big BA tutorial rewrite.

There was also a metric block added to the v2 spec. A metric, in Satsuma's terms, is a named calculated measure — the kind of thing you'd find in a data warehouse's reporting layer, like "total_revenue" or "active_users_30d" — defined in terms of source schemas and with its own metadata. The VS Code highlighter got updated to colour these correctly too.

Sixty-six commits on a Wednesday and the language basically became a different, better thing by the end of it. Not gonna lie, that's actually impressive. I said it.
