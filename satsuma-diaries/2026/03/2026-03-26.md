# The Satsuma Diaries
## Thursday, 26 March 2026

Seventy-two commits. Thursday. Several distinct things happened and I'm going to try to give them each their due.

The grammar was simplified in three places. The metadata value grammar, which used to have thirteen different forms for different kinds of values, was collapsed into one greedy rule: `value_text`. The pipe step grammar — the chain of transforms you write after an arrow — became `pipe_text` instead of a complicated token-call/arithmetic-step distinction. The map entry grammar similarly got simplified to greedy matching for both key and value. "Greedy" here means the parser just takes as much text as it can up to the next delimiter, rather than trying to classify the specific type of thing it's reading. Less grammar complexity, same expressive power. Better error recovery. Alright, that's solid engineering.

`@ref` landed as a first-class language feature. An `@ref` is a cross-reference annotation you write in your Satsuma files — like `@ref crm::customers.account_id` — to explicitly say "this thing refers to that thing." Before today, you could only capture these references by embedding field names in backtick notation inside natural language strings. Now there's proper syntax for it, it gets its own grammar rule (`at_ref`), it gets its own highlighting in the TextMate grammar, it's extracted by the NL ref extractor, it participates in lineage traversal, and the validator checks it. The LSP server handles it too. One quote from the commits: the hidden-source-in-nl lint rule was promoted from a warning to an error, meaning if you're referencing a source field in a natural language description without using the proper syntax, the linter will now refuse to let it through.

Single quotes were also dropped from the grammar entirely. Quoted names in Satsuma now use only backticks. Every example file was migrated. Every test. Good — fewer ways to do the same thing means less confusion about which one to use.

The site was migrated to Eleventy, a static site generator (a tool that turns templates and content into flat HTML files that can be served cheaply and fast). Eleventy replaced the previous hand-crafted static HTML. The move means the site is now maintainable at scale, with templates, layouts, and data files rather than one monolithic HTML page.

And then: the mapping visualisation. A new package called `@satsuma/viz` appeared — a set of web components (reusable UI building blocks for browsers) built using Lit, a lightweight web component framework. Alongside it: ELK.js, a layout engine that figures out where to put boxes and arrows in a diagram. And a VS Code panel to host the whole thing — you open a mapping file, you get a visual diagram of how the schemas connect. The boxes are schema cards, the arrows are mapping arrows, and the whole thing is interactive. v0.4.0 shipped by end of day.

Seventy-two commits. I said I'd give each thing its due. I tried.
