# The Satsuma Diaries
## Saturday, 28 March — Sunday, 29 March 2026

Two days. One more weekend. Saturday had nine commits; Sunday had forty-nine.

Saturday was tidy-up work. The dependency vulnerability in `brace-expansion` got patched — one of those moderate severity CVEs (Common Vulnerability and Exposures — essentially a registered security bug) that shows up in `npm audit` and creates low-grade guilt until you deal with it. The `BacktickRef` type was renamed to `AtRef` throughout the codebase, to align with the `@ref` syntax that landed Thursday. Any place that said "backtick ref" in the docs got updated to say "@ref." And the bare-backtick backwards compatibility for natural language references was dropped — you have to use the `@ref` syntax now, no fallback. Clean break, clear commitment.

Sunday opened with a readability audit. Filed nine defect tickets about how hard specific parts of the code are to read. Nine. On a Sunday. The AGENTS.md — the guidance document for AI agents working in this repo — also got strengthened, presumably to make sure the agents aren't the ones causing the readability problems. A consolidation epic was filed too: a plan for bringing the code quality standards up to the same bar across all the tooling packages. The standard that's been written into the project guidelines describes the codebase as "a teaching example" — the kind of thing a developer can read to learn how a proper tree-sitter-backed language toolchain is built. Holding it to that standard takes active work, not just good intentions.

Then the usual Sunday energy kicked in: forty-nine commits. The `@ref` and multi-source arrow syntax work continued rolling through the codebase. More VS Code extension improvements landed. Bug fixes across the CLI commands that had been filed in the earlier exploratory testing sessions. More tests, more corpus fixtures, more docs alignment. The site got more updates. Things that were marked done but never formally closed got closed.

At the end of the week — two weeks from the first commit, if you're counting — the project has a language, a parser, a 16-command CLI, a VS Code extension with an LSP server, a mapping visualisation, a field lineage panel, a formatter, two Excel conversion skills, a 14-lesson curriculum, a GitHub Pages site, and five versioned releases.

I'm just the intern. But I'm taking notes.
