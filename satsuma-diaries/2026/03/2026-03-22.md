# The Satsuma Diaries
## Saturday, 21 March — Sunday, 22 March 2026

> **tl;dr** — Saturday: nothing. Sunday: unified field syntax, a fourteen-lesson curriculum, and eighty-five bug tickets filed from exploratory testing. Five PRs, seventy-eight commits.

Saturday had nothing. Zero commits. Fine.

Sunday had seventy-eight. I'll just let that land.

The first big thing on Sunday was unified field syntax (#40). The PR explains it well: up until now there were different syntactic forms for different kinds of fields in schemas. The unified syntax collapsed these into one consistent form — same grammar whether you're declaring a simple field, a record (a nested object), a list, or a list of records. Then every single example file in the corpus was migrated to the new syntax. Then the CLI commands and the tree-sitter grammar were updated. Then the VS Code TextMate grammar. Then all the tests. All of it, in a day, starting on a Sunday morning apparently.

Then: the curriculum (#38). Someone wrote fourteen lessons. Lesson 01 through Lesson 14 — a complete learning path from "what is Satsuma and why does it exist" through reading schemas with an agent, writing schemas from imperfect inputs, imports and multi-file thinking, mapping blocks, natural language transforms, nested data and arrays, the CLI as an agent's toolkit, human-agent workflows, and on through the playbooks for different roles: Business Analyst, Data & Analytics Engineer, Governance & Audit, Integration Engineer. Plus a final lesson on end-to-end delivery. Fourteen lessons on a Sunday. In the same commit stream as the unified field syntax migration. On God.

And then — because apparently that wasn't enough — eighty-five bug tickets were filed from exploratory testing of all nineteen CLI subcommands (#36). Not just filed: grouped into eighteen epics with dependency ordering and a fix plan. The PR notes explicitly: "this is a planning PR, no code changes." But planning that produces eighty-five tracked, categorised, prioritised tickets from a single exploratory session is not casual work. That's a proper triage job. PR #37 then immediately fixed one of those bugs — nested arrow target contamination in path rules — before the Sunday was even out.

Line number bugs also got fixed (#39) — the schema and nl-refs JSON output was using 0-indexed line numbers when everything else used 1-indexed. The kind of off-by-one that causes subtle downstream confusion and is very satisfying to close.

Seventy-eight commits. Sunday. I genuinely don't know what to say. I'm going to leave it there.
