# The Satsuma Diaries
## Friday, 27 March 2026

> **tl;dr** — Shared code was extracted into a proper library so the CLI, VS Code extension, and visualiser stop doing the same work independently. The field lineage panel landed in VS Code. Examples got reorganised. Thirteen PRs, thirty-two commits.

Thirty-two commits. Friday. The consolidation work landed today, and it's the kind of thing that doesn't make headlines but makes everything better.

`satsuma-core` became a real package (#125). Previously, the CLI tool, the LSP server, and the visualisation component were all doing their own versions of the same fundamental operations — extracting field declarations from the syntax tree, expanding fragment spreads, resolving canonical names for references. PR #125 pulled those out into a shared library called `@satsuma/core` which all three consumers can now import. The description is straightforward: "extract the shared formatter and supporting types into a new tooling/satsuma-core package." The CLI shims (temporary compatibility wrappers that existed while the migration was being done) were deleted. Clean.

But before the day was out there were already follow-up release fixes. PR #116 fixed the universal CLI build and a VSIX CI failure. PR #119 fixed the CLI release packaging for a missing parser WASM file. PR #121 set the executable permission on the CLI entry point — `tsc` emits `dist/index.js` with 644 permissions, meaning not executable, and the release tarball captures that, and `npm install -g` preserves it, causing "permission denied" on first run. PR #124 fixed the same problem for `npm pack` specifically, because `postbuild` only fires after `npm run build` and `npm pack` triggers `prepare` instead, bypassing the hook. Four PRs to get a release working correctly across platforms. That's just how it goes sometimes.

The mapping visualisation got several fixes (#117, #126). PR #126 addressed bounds and fit calculations so namespace boxes and content extents match the rendered cards. The minimap anchoring got fixed too — it was wrong on window resize. ELK node sizing for larger metric and report cards was adjusted to reduce overlap. The kind of layout work that requires patience and a good eye.

v0.4.0 was officially prepared (#118). Seven tickets that had been delivered but never formally closed were swept up (#120). The AI agent reference doc was improved and synced across the whole repo — 37 files updated to remove bogus token counts and add missing conventions (#122). Context-aware lineage actions landed in the LSP server (#123) — clickable "Lineage from / Lineage to" code lenses that prefill from the schema and field the cursor is already on, rather than requiring you to type the name manually.

v0.5.0 shipped. That's the fifth versioned release in seven working days. I'm just going to note that and move on.
