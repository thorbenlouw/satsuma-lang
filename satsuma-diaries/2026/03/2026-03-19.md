# The Satsuma Diaries
## Thursday, 19 March 2026

> **tl;dr** — Namespaces landed across the whole toolchain, natural language field references became machine-readable, the graph export command shipped, and releases got automated. Eleven PRs, thirty-two commits.

Thirty-two commits. Thursday. Coming down from the madness of Wednesday, which is fair.

The theme today was namespaces — which is exactly what it sounds like, a way of giving things names that don't collide with each other. In a big data platform you might have five different teams all working on their own pipeline files, and they've all got a schema called `customers`. Without namespaces, these clash horribly and nobody knows whose `customers` is whose. With namespaces, you write `crm::customers` or `billing::customers` and now it's clear. PR #18 landed the namespace support in the grammar, the extraction logic, the index builder, the validator, and every one of the CLI commands. PRs #19 and #20 then fixed six namespace bugs on the same day — which means someone tested it properly rather than just shipping and hoping.

Also shipped via #20: NL backtick references. NL is Natural Language — the plain English descriptions you can write in Satsuma to explain what a mapping is doing. Until today, these were just text. From today, you can embed a field reference inside those descriptions using backtick notation, like the value comes from `crm::customers.account_id` after normalisation. The system can now extract these references and include them in lineage analysis (lineage is the trail of where your data came from and where it goes). That means human-readable notes become machine-readable too. Both audiences served at once.

The `stm graph` command also appeared (#22) — a way to export the semantic graph of your whole workspace to JSON so other tools can consume it. A semantic graph is basically a map of all the schemas, mappings, and relationships in your project, expressed as nodes and edges rather than files. Useful if you want to visualise the whole thing or feed it into another system. The PR also added a feature audit and created tickets for remaining work, which is the kind of organised thinking that keeps a fast-moving project coherent.

There was also a release workflow (#13), which means GitHub will now automatically build and attach the compiled binaries when a new version tag is pushed. And some additional parser quirks got resolved (#15) — triple-quoted strings with embedded quotes, for instance, which is one of those things that sounds simple and definitely isn't.

SAP PO examples and parser fixtures arrived too (#16). SAP Process Orchestration is a fairly gnarly enterprise integration platform — the fact that Satsuma has specific test cases for mapping from it tells you this is targeting real industrial pipelines, not just tidy academic examples.

Solid Thursday. Grounded, practical, no drama.
