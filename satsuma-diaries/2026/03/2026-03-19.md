# The Satsuma Diaries
## Thursday, 19 March 2026

Thirty-two commits. Thursday. Coming down from the madness of Wednesday, which is fair.

The theme today was namespaces — which is exactly what it sounds like, a way of giving things names that don't collide with each other. In a big data platform you might have five different teams all working on their own pipeline files, and they've all got a schema called `customers`. Without namespaces, these clash horribly and nobody knows whose `customers` is whose. With namespaces, you write `crm::customers` or `billing::customers` and now it's clear. Today that support landed in the grammar, the extraction logic, the index builder, the validator, and every one of the CLI commands. Six namespace bugs were also found and fixed on the same day, which means someone tested it properly rather than just shipping and hoping.

Also shipped: NL backtick references. NL is Natural Language — the plain English descriptions you can write in Satsuma to explain what a mapping is doing. Until today, these were just text. From today, you can embed a field reference inside those descriptions using backtick notation, like ``the value comes from `crm::customers.account_id` after normalisation``. The system can now extract these references and include them in lineage analysis (lineage is the trail of where your data came from and where it goes — think of it as the paper trail for your pipeline). That means human-readable notes become machine-readable too. Both audiences served at once.

The `stm graph` command also appeared — a way to export the semantic graph of your whole workspace to JSON so other tools can consume it. A semantic graph is basically a map of all the schemas, mappings, and relationships in your project, expressed as nodes and edges rather than files. Useful if you want to visualise the whole thing or feed it into another system.

There was also a release workflow, which means GitHub will now automatically build and attach the compiled binaries when a new version tag is pushed. And some additional parser quirks got resolved — triple-quoted strings with embedded quotes, for instance, which is one of those things that sounds simple and definitely isn't.

Solid Thursday. Grounded, practical, no drama.
