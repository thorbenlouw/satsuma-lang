# The Satsuma Diaries
## Monday, 23 March 2026

> **tl;dr** — The VS Code extension got a full language server in one day: go-to-definition, find-all-references, completions, hover, CodeLens, rename, and more. Fifteen PRs, forty-five commits.

Forty-five commits. Monday. The VS Code extension grew a brain today.

Up until now, the VS Code extension did syntax highlighting — colours. Nice, but passive. Today it got an LSP server. LSP stands for Language Server Protocol — it's a standardised way for an editor to talk to a program that actually understands a language, so you get proper IDE features: go-to-definition, find-all-references, hover information, completions, error diagnostics in real time. The protocol is the reason you can get the same smart features in VS Code, JetBrains, Neovim, and anything else that speaks LSP — the language server only needs to be written once.

Three PRs did the heavy lifting. LSP Phase 1 (#47) built the scaffold: document symbols (the outline panel that shows you what's in a file), code folding (collapse a mapping block with a click), diagnostic errors from running `satsuma validate` behind the scenes, semantic tokens (which gives the highlighter more context — it knows a field reference is a field reference specifically, not just a word that happens to look like one), and hover (so when you mouse over a schema name you get a tooltip with actual information). Phase 2 (#50) added go-to-definition, find-all-references, and completions. Phase 3 (#51) added CodeLens — those little action hints that appear above blocks of code (things like "Show Lineage", "3 references") — and Rename Symbol: rename a schema in one place and watch it update everywhere. Plus custom workspace-level commands for graph export, field lineage, and mapping coverage, baked into the command palette.

The `.vsix` build — the actual VS Code extension package, the file you install — was automated into CI (#52), so releases produce a real installable file. There was some additional vsix build work and platform targets added (#53) to get it working across Mac, Linux, and Windows.

The `satsuma agent-reference` command also shipped (#41) — a CLI command that outputs the compact AI Agent Reference, so AI agents running in a terminal can pull the language grammar and conventions without needing to read a project file. Self-referential, but practical. And twenty-two bug tickets got closed (#42) that had already been fixed by prior work but nobody had marked as done. Someone did a sweep. Good hygiene.

The Excel conversion tooling also saw progress: the Excel-to-STM prompt was moved to its proper home in `useful-prompts/` (#43), and a Satsuma-to-Excel lite prompt appeared alongside it. Both directions covered, no special tooling required.

Completed features got archived (#44) — feature docs that had shipped were moved out of the active `features/` folder into archive, so the active roadmap only shows what's actually open. That's the kind of quiet housekeeping that keeps a project readable.
