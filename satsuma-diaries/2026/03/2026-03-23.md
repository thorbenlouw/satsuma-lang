# The Satsuma Diaries
## Monday, 23 March 2026

Forty-five commits. Monday. The VS Code extension grew a brain today.

Up until now, the VS Code extension did syntax highlighting — colours. Nice, but passive. Today it got an LSP server. LSP stands for Language Server Protocol — it's a standardised way for an editor to talk to a program that actually understands a language, so you get proper IDE features: go-to-definition, find-all-references, hover information, completions, error diagnostics in real time. The protocol is the reason you can get the same smart features in VS Code, JetBrains, Neovim, and anything else that speaks LSP — the language server only needs to be written once.

The Satsuma LSP server was built today in stages. First: the scaffold — document symbols (the outline panel that shows you what's in a file), code folding (collapse a mapping block with a click), and diagnostic errors from running `satsuma validate` behind the scenes. Then semantic tokens, which gives the highlighter more context than the TextMate grammar can provide — it knows a field reference is a field reference specifically, not just a word that happens to look like one. And hover — so when you mouse over a schema name you get a tooltip with information about it.

Then go-to-definition landed: click on a schema name, jump to where it's defined. Find-all-references: right-click a field, see everywhere it's used. Completions: start typing and get suggestions. These are the features that make a language feel like a real language in your editor, not just a text file with colours.

Then CodeLens — those little action hints that appear above certain blocks of code (things like "Show Lineage", "3 references"). And Rename Symbol: rename a schema in one place and watch it update everywhere.

And to top it all off: custom workspace-level commands for graph export, field lineage, and mapping coverage, baked into the command palette. And the `.vsix` build — the actual VS Code extension package — was automated into CI, so releases produce a real installable file.

Twenty-two bug tickets also got closed today, the ones that had already been fixed by prior work but nobody had marked as done. Someone did a sweep. Good hygiene.
