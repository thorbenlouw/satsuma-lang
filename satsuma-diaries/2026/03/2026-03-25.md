# The Satsuma Diaries
## Wednesday, 25 March 2026

Twenty-eight commits. Wednesday. Relative calm, by recent standards.

The main feature was multi-source arrows — the ability for a single mapping arrow to declare that it draws from more than one source field, rather than just one. The grammar was extended to support this, tests were written, the extraction logic was updated, the CLI was updated, and the spec documentation was brought in line. When you're mapping data from multiple systems, you often need to combine fields from several places to produce one output; now Satsuma has first-class syntax for saying so.

Alongside that: a `canonicalRef()` utility function. This is a helper that produces a consistent, fully-qualified name for a field reference regardless of how it was written in the source file — `customers.id`, `crm::customers.id`, and a backtick-quoted version all resolve to the same canonical string. Useful when you're trying to match things up across files or compare references that might have been written slightly differently. This is the kind of unsexy internal infrastructure that makes everything else more reliable.

The integration tests were also parallelised today — the test suite went from taking 24 seconds to 7.5 seconds. Not a feature anyone will notice from the outside, but agents running the test suite many times per hour will notice. Compounding savings.

The Satsuma-to-Excel skill was also added — the counterpart to the Excel-to-Satsuma skill — using a deterministic Python tool. Deterministic here means: given the same input, you always get the same output, no randomness, no variation. Good property for a conversion tool.

v0.3.0 was also bumped in here somewhere. Third versioned release. The changelog landed, the site version was updated. The docs got aligned with the new `@ref` syntax across lessons, convention guides, and the BA tutorial. Small but consistent signals that someone is thinking about coherence across the whole surface area.
