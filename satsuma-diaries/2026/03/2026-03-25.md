# The Satsuma Diaries
## Wednesday, 25 March 2026

> **tl;dr** — The Satsuma-to-Excel export skill shipped, multi-source arrows landed as a first-class language feature, the test suite got three times faster, and the language simplification work started. Seven PRs, twenty-eight commits.

Twenty-eight commits. Wednesday. Relative calm, by recent standards.

The Satsuma-to-Excel skill shipped (#97) — the counterpart to the Excel-to-Satsuma skill. You can now go in both directions using AI tooling: hand an agent a spreadsheet and get back Satsuma files, or hand it Satsuma files and get back a spreadsheet. The PR uses a deterministic Python tool — deterministic meaning: given the same input, you always get the same output, no randomness, no variation. Good property for a conversion tool. This wraps up the bidirectional Excel story that started on day four.

Multi-source arrows landed via the language simplification work (#103). The feature: a single mapping arrow can now declare that it draws from more than one source field, rather than just one. The PR covers the whole surface — grammar extended, tests written, extraction logic updated, CLI updated, spec documentation aligned. When you're mapping data from multiple systems, you often need to combine fields from several places to produce one output; now Satsuma has first-class syntax for saying so, rather than working around it.

Alongside that: a `canonicalRef()` utility function. This is a helper that produces a consistent, fully-qualified name for a field reference regardless of how it was written in the source file — `customers.id`, `crm::customers.id`, and a backtick-quoted version all resolve to the same canonical string. Useful when you're trying to match things up across files or compare references that might have been written slightly differently. The unsexy internal infrastructure that makes everything else more reliable.

The integration tests were also parallelised today, somewhere in PRs #98–#100. The test suite went from twenty-four seconds to seven-and-a-half seconds. Not a feature anyone will notice from the outside, but agents running the test suite many times per hour will notice. Compounding savings.

Three bug-fix PRs also merged (#98, #99, #100) — version output, formatter edge cases, deeply nested paths in natural language references, nl truncation removed, and eighteen more CLI bugs across the board from the batch-3 exploratory testing sweep. Someone had a long list and was working through it methodically.

v0.3.0 was bumped in here somewhere. Third versioned release. The changelog landed, the site version was updated. The docs got aligned with the new `@ref` syntax across lessons, convention guides, and the BA tutorial. Small but consistent signals that someone is thinking about coherence across the whole surface area.
