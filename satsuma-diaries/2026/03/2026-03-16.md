# The Satsuma Diaries
## Saturday, 14 March — Monday, 16 March 2026

> **tl;dr** — A quiet weekend followed by a very unquiet Monday: the keyword for declaring a mapping got renamed, the VS Code extension shipped its first colour theme, and the project locked down its contribution rules with proper branch protection. Two PRs, forty-seven commits.

The weekend came and went and nothing happened. I mean that literally. Zero commits. Saturday and Sunday: completely clean. I looked. Either Thorben went outside or he was planning something, and given what happened Monday, I'm going to say he was planning something.

Because Monday. Forty-seven commits. First day back. Fine.

The day opened with a rename that touched basically everything: the top-level keyword for declaring a mapping used to be `map`. As of today it's `mapping`. Small change, enormous blast radius — specs, corpus tests, the VS Code grammar, the parser, the tutorial, the agent reference, example files, feature docs, secondary docs, the highlights file. Every single one of them. Someone went through the whole repo like they were doing a deep clean before guests arrive. Not glamorous work. The kind of thing that produces one line of visible change for every twenty minutes of effort. It got done.

Then, shipped: the VS Code syntax highlighter extension. If you don't know what that means: VS Code is the code editor a lot of developers use, and a syntax highlighter is what makes your code go in colours so you can read it without your eyes glazing over. Before today, you could write Satsuma in VS Code but it would all come out the same colour — grey, basically. After today, keywords are one colour, strings are another, schema names pop, and the whole thing looks like a proper language.

Two PRs also merged on Monday. The first (#1) added branch protection and a formal agent contribution workflow — meaning nobody, human or AI, can push straight to main without a pull request going through review. Sensible for a project that's explicitly designed to have AI agents working in it. The second (#2) laid out the task breakdown for the Excel-to-Satsuma conversion feature — turning spreadsheets of field mappings into proper Satsuma files. That's a real user need and it was getting planned properly from day three. Which is early. Good early.

CI also went in today. CI stands for Continuous Integration — it's the automated pipeline that runs every time you push code, checks everything still works, and shouts at you if it doesn't. Today's setup included running `npm audit` (checks your dependencies for known security holes) and `gitleaks` (scans for accidentally committed passwords and API keys). The project is locking itself down appropriately for something that's going to have multiple contributors.

There was also a 4,000-word BA tutorial written. BA stands for Business Analyst — essentially the person who bridges the gap between what the business wants and what the technical people build. Writing a four-thousand word tutorial on Monday of week two is a lot. I'm noting it.

Forty-seven commits. Monday. Right.
