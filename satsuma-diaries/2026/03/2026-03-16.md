# The Satsuma Diaries
## Saturday, 14 March — Monday, 16 March 2026

The weekend came and went and nothing happened. I mean that literally. Zero commits. Saturday and Sunday: completely clean. I looked. Either Thorben went outside or he was planning something, and given what happened Monday, I'm going to say he was planning something.

Because Monday. Forty-seven commits. First day back. Fine.

The day opened with a rename that touched basically everything: the top-level keyword for declaring a mapping used to be `map`. As of today it's `mapping`. Small change, enormous blast radius — specs, corpus tests, the VS Code grammar, the parser, the tutorial, the agent reference, example files, feature docs, secondary docs, the highlights file. Every single one of them. Someone went through the whole repo like they were doing a deep clean before guests arrive. Not glamorous work. The kind of thing that produces one line of visible change for every twenty minutes of effort. It got done.

Then, shipped: the VS Code syntax highlighter extension. If you don't know what that means: VS Code is the code editor a lot of developers use, and a syntax highlighter is what makes your code go in colours so you can read it without your eyes glazing over. Before today, you could write Satsuma in VS Code but it would all come out the same colour — grey, basically. After today, keywords are one colour, strings are another, schema names pop, and the whole thing looks like a proper language. That's `feat(stm-t1n8)` if you're tracking the tickets. One commit, but it's not nothing.

CI also went in today. CI stands for Continuous Integration — it's the automated pipeline that runs every time you push code, checks everything still works, and shouts at you if it doesn't. Today's setup included running `npm audit` (checks your dependencies for known security holes) and `gitleaks` (scans for accidentally committed passwords and API keys). Branch protection was also added, which means nobody can push directly to the main branch without a pull request. The project is locking itself down appropriately for something that's going to have multiple contributors, including AI agents who sometimes get confident ideas.

There was also a 4,000-word BA tutorial written. BA stands for Business Analyst — essentially the person who bridges the gap between what the business wants and what the technical people build. Writing a four-thousand word tutorial is a lot of effort for a Monday. There were also early tickets and a PRD (Product Requirements Document — the "here's what we're building and why" doc before anyone writes any code) for Excel-to-Satsuma conversion. The idea: take a spreadsheet with field mappings in it and convert it to proper Satsuma files automatically. Sounds useful. Sounds like future work.

Forty-seven commits. Monday. Right.
